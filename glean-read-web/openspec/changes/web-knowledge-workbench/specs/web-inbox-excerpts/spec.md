# 规范：web-inbox-excerpts

## 目的
定义 GleanRead Web 端左侧收件箱侧栏的摘录筛选、卡片展示和拖拽挂载规则，确保用户能够将未挂载摘录快速整理到知识树节点。

## 需求

### 需求:收件箱必须默认展示未挂载且未删除的摘录
系统必须在收件箱侧栏默认展示满足 `tree_node_id IS NULL AND is_deleted = false` 的摘录，并提供切换到“全部摘录”的入口。切换到“全部摘录”后，系统必须展示当前用户全部未删除摘录，但不得把已删除摘录重新纳入列表。

#### 场景:首次打开收件箱
- **当** 用户首次进入主工作台且左侧收件箱可见
- **那么** 系统必须默认列出未挂载且未删除的摘录
- **那么** 系统不得默认混入已挂载摘录

#### 场景:切换到全部摘录
- **当** 用户在收件箱中切换到“全部摘录”视图
- **那么** 系统必须显示当前用户所有未删除摘录
- **那么** 系统不得显示 `is_deleted = true` 的记录

#### 场景:收件箱为空
- **当** 当前筛选条件下没有可展示摘录
- **那么** 系统必须显示空状态说明
- **那么** 空状态必须保留收件箱侧栏的过滤入口

### 需求:收件箱卡片必须展示摘录核心信息
系统必须在收件箱卡片中展示摘录正文、来源信息、我的思考和标签。若 `source_title` 为空，系统必须使用 `url` 作为来源展示；若 `user_thought` 为空，系统必须以弱提示代替空白；标签必须按当前关联关系展示，并可使用颜色或图标区分。

#### 场景:来源标题为空
- **当** 某条摘录的 `source_title` 为空但 `url` 存在
- **那么** 系统必须在卡片中展示 `url`
- **那么** 系统不得留下空白来源标题区域

#### 场景:我的思考为空
- **当** 某条摘录的 `user_thought` 为空
- **那么** 系统必须在卡片中显示弱提示
- **那么** 系统不得以空白区域替代提示

#### 场景:展示标签
- **当** 某条摘录存在一个或多个标签关联
- **那么** 系统必须在卡片中展示对应标签
- **那么** 标签展示必须与当前摘录绑定关系一致

### 需求:收件箱摘录必须支持拖拽挂载到知识树节点
系统必须允许用户从收件箱卡片拖拽摘录到中间知识树画布中的节点。拖拽经过某个节点时，该节点必须呈现高亮反馈；用户松开鼠标后，系统必须将该摘录的 `tree_node_id` 更新为目标节点 ID，并在本地工作台中即时反映挂载结果。

#### 场景:拖拽经过节点
- **当** 用户拖拽某条收件箱摘录悬停在某个知识树节点上方
- **那么** 系统必须高亮该节点以表示可挂载目标

#### 场景:拖拽释放到节点
- **当** 用户松开鼠标并将摘录释放到目标节点上
- **那么** 系统必须把该摘录的 `tree_node_id` 更新为目标节点 ID
- **那么** 该摘录必须从未挂载收件箱视图中消失
- **那么** 目标节点的摘录数量必须立即加一

#### 场景:拖拽释放到无效区域
- **当** 用户将摘录拖出有效节点区域后松开
- **那么** 系统不得修改该摘录的挂载位置
- **那么** 卡片必须保持在原来的收件箱列表中
