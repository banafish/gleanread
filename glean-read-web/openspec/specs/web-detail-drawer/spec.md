# 规范：web-detail-drawer

## Purpose
定义 GleanRead Web 端右侧详情沉淀抽屉的内容结构、Markdown 编辑、摘录流展示和标签交互规则，确保用户能够围绕当前知识树节点完成阅读、整理和沉淀。

## Requirements
### Requirement: 详情抽屉必须随当前选中节点展示
系统 MUST在用户选中知识树节点时展示右侧详情抽屉，并把当前节点作为抽屉内容的上下文。抽屉顶部必须展示当前节点标题，并提供全屏专注写作入口；当用户未选中节点时，系统 MUST显示空态或引导态，而不是显示错误内容。

#### Scenario: 选中节点后打开抽屉
- **WHEN** 用户在知识树画布中点击某个节点
- **THEN** 系统 MUST打开右侧详情抽屉并切换到该节点上下文
- **THEN** 抽屉顶部必须显示该节点标题

#### Scenario: 未选中节点
- **WHEN** 当前没有选中任何知识树节点
- **THEN** 系统 MUST显示抽屉空态或引导态
- **THEN** 系统不得显示旧节点内容

#### Scenario: 进入专注写作模式
- **WHEN** 用户点击抽屉顶部的全屏展开入口
- **THEN** 系统 MUST将抽屉切换为全屏或覆盖式专注写作模式
- **THEN** 用户退出后必须回到原来的抽屉布局

### Requirement: 抽屉顶部必须提供节点 Markdown 沉淀编辑器
系统 MUST在抽屉顶部提供与当前节点 `outline_markdown` 绑定的 Markdown 编辑器，并支持块级编辑与快捷输入。编辑器必须允许用户直接输入、修改和组织大纲内容，且当内容为空时显示适当的占位提示。

#### Scenario: 编辑已有大纲
- **WHEN** 当前节点已存在 `outline_markdown`
- **THEN** 系统 MUST在编辑器中显示该内容
- **THEN** 用户修改后必须能继续在同一位置编辑

#### Scenario: 编辑空大纲
- **WHEN** 当前节点的 `outline_markdown` 为空
- **THEN** 系统 MUST展示空白编辑态与占位提示
- **THEN** 系统不得隐藏编辑器本身

#### Scenario: 使用块级快捷输入
- **WHEN** 用户在编辑器中输入 `/`
- **THEN** 系统 MUST唤出块级编辑菜单或等价入口
- **THEN** 用户应能快速插入常见 Markdown 块

### Requirement: 抽屉底部必须展示当前节点的摘录流
系统 MUST在抽屉底部展示所有挂载到当前节点的摘录，并允许用户围绕单条摘录进行快速整理。摘录卡片必须展示内容、来源、我的思考和标签；当当前节点没有挂载摘录时，系统 MUST展示空状态，而不是占位列表。

#### Scenario: 当前节点有挂载摘录
- **WHEN** 某个摘录的 `tree_node_id` 等于当前选中节点 ID
- **THEN** 系统 MUST在抽屉底部展示该摘录
- **THEN** 该摘录必须与当前节点上下文一致

#### Scenario: 当前节点无摘录
- **WHEN** 当前节点没有任何挂载摘录
- **THEN** 系统 MUST显示摘录空状态
- **THEN** 系统不得展示错误的其他节点摘录

#### Scenario: 快速编辑我的思考
- **WHEN** 用户点击摘录卡片中的“我的思考”区域并输入内容后失焦
- **THEN** 系统 MUST保存该摘录的 `user_thought`
- **THEN** 用户再次打开该卡片时必须看到最新内容

### Requirement: 摘录卡片必须支持标签补全与移回未分类
系统 MUST在摘录卡片中展示当前标签，并提供新增或选择标签的补全入口；卡片菜单必须提供“移回未分类”动作，将该摘录的 `tree_node_id` 设回 `NULL`。

#### Scenario: 添加标签
- **WHEN** 用户点击摘录卡片中的标签添加入口
- **THEN** 系统 MUST打开标签自动补全或新建入口
- **THEN** 用户确认后必须更新该摘录的标签关联

#### Scenario: 移回未分类
- **WHEN** 用户在摘录卡片菜单中点击“移回未分类”
- **THEN** 系统 MUST把该摘录的 `tree_node_id` 设为 `NULL`
- **THEN** 该摘录必须从当前节点的摘录流中消失
