# 规范：web-inbox-excerpts

## Purpose
定义 GleanRead Web 端左侧收件箱侧栏的摘录筛选、卡片展示和拖拽挂载规则，确保用户能够将未挂载摘录快速整理到知识树节点。

## Requirements
### Requirement: 收件箱必须默认展示未挂载且未删除的摘录
系统 MUST在收件箱侧栏默认展示满足 `tree_node_id IS NULL AND is_deleted = false` 的摘录，并提供切换到“全部摘录”的入口。切换到“全部摘录”后，系统 MUST展示当前用户全部未删除摘录，但不得把已删除摘录重新纳入列表。

#### Scenario: 首次打开收件箱
- **WHEN** 用户首次进入主工作台且左侧收件箱可见
- **THEN** 系统 MUST默认列出未挂载且未删除的摘录
- **THEN** 系统不得默认混入已挂载摘录

#### Scenario: 切换到全部摘录
- **WHEN** 用户在收件箱中切换到“全部摘录”视图
- **THEN** 系统 MUST显示当前用户所有未删除摘录
- **THEN** 系统不得显示 `is_deleted = true` 的记录

#### Scenario: 收件箱为空
- **WHEN** 当前筛选条件下没有可展示摘录
- **THEN** 系统 MUST显示空状态说明
- **THEN** 空状态必须保留收件箱侧栏的过滤入口

### Requirement: 收件箱卡片必须展示摘录核心信息
系统 MUST在收件箱卡片中展示摘录正文、来源信息、我的思考和标签。若 `source_title` 为空，系统 MUST使用 `url` 作为来源展示；若 `user_thought` 为空，系统 MUST以弱提示代替空白；标签必须按当前关联关系展示，并可使用颜色或图标区分。

#### Scenario: 来源标题为空
- **WHEN** 某条摘录的 `source_title` 为空但 `url` 存在
- **THEN** 系统 MUST在卡片中展示 `url`
- **THEN** 系统不得留下空白来源标题区域

#### Scenario: 我的思考为空
- **WHEN** 某条摘录的 `user_thought` 为空
- **THEN** 系统 MUST在卡片中显示弱提示
- **THEN** 系统不得以空白区域替代提示

#### Scenario: 展示标签
- **WHEN** 某条摘录存在一个或多个标签关联
- **THEN** 系统 MUST在卡片中展示对应标签
- **THEN** 标签展示必须与当前摘录绑定关系一致

### Requirement: 收件箱摘录必须支持拖拽挂载到知识树节点
系统 MUST允许用户从收件箱卡片拖拽摘录到中间知识树画布中的节点。拖拽经过某个节点时，该节点必须呈现高亮反馈；用户松开鼠标后，系统 MUST将该摘录的 `tree_node_id` 更新为目标节点 ID，并在本地工作台中即时反映挂载结果。

#### Scenario: 拖拽经过节点
- **WHEN** 用户拖拽某条收件箱摘录悬停在某个知识树节点上方
- **THEN** 系统 MUST高亮该节点以表示可挂载目标

#### Scenario: 拖拽释放到节点
- **WHEN** 用户松开鼠标并将摘录释放到目标节点上
- **THEN** 系统 MUST把该摘录的 `tree_node_id` 更新为目标节点 ID
- **THEN** 该摘录必须从未挂载收件箱视图中消失
- **THEN** 目标节点的摘录数量必须立即加一

#### Scenario: 拖拽释放到无效区域
- **WHEN** 用户将摘录拖出有效节点区域后松开
- **THEN** 系统不得修改该摘录的挂载位置
- **THEN** 卡片必须保持在原来的收件箱列表中
