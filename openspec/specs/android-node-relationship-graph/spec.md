# 规范：android-node-relationship-graph

## 目的
定义 Android 节点详情页中的局部关系图能力，保证图谱完全由本地关联数据构建并支持交互导航。

## 需求

### 需求:节点详情页必须提供可交互的局部关系图入口
系统必须在节点详情页提供与 `doc/ui.tsx` 对应的局部图谱入口，并展示当前节点为中心的局部关系图视图。

#### 场景:打开局部图谱
- **当** 用户在节点详情页点击右上角的局部图谱按钮
- **那么** 系统必须打开当前节点的局部关系图，并把当前节点显示为中心节点

### 需求:局部关系图必须基于本地关联数据构建
系统必须仅根据本地知识树节点、挂载摘录和双向链接解析结果构建局部图谱，并区分不同类型的关系节点。

#### 场景:构建局部图谱数据
- **当** 系统为某个节点生成局部图谱
- **那么** 图谱中必须至少包含当前节点、该节点挂载的摘录、当前节点引用的节点以及引用当前节点的节点，并使用不同视觉样式区分节点与摘录

#### 场景:本地更新后刷新图谱
- **当** 当前节点的摘录挂载关系或双向链接在本地发生变化
- **那么** 系统必须在重新打开图谱或主动刷新图谱时反映最新的本地关系

### 需求:局部关系图必须支持手势与点击交互
系统必须允许用户在图谱视图中执行拖拽、缩放、点击聚焦和本地导航/预览操作。

#### 场景:拖拽和缩放图谱
- **当** 用户在图谱视图中拖拽或双指缩放
- **那么** 系统必须允许平移和缩放图谱，而不得关闭当前图谱视图

#### 场景:点击图谱中的知识节点
- **当** 用户点击图谱中的某个知识节点
- **那么** 系统必须聚焦该节点，并允许用户跳转到该节点的详情页

#### 场景:点击图谱中的摘录节点
- **当** 用户点击图谱中的某个摘录节点
- **那么** 系统必须打开该摘录的本地预览或详情内容
