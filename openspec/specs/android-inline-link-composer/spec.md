# 规范：android-inline-link-composer

## 目的
定义 Android 端 `[[` 双向链接输入、结构化存储和展示态跳转规则，确保本地联想与本地导航体验一致。

## 需求

### 需求:`[[` 输入必须触发本地联想面板
系统必须在摘录想法输入框和节点大纲编辑框中监听 `[[` 触发，并基于本地知识节点标题与摘录内容实时展示联想面板。

#### 场景:输入 `[[` 后搜索节点与摘录
- **当** 用户在支持联想的输入框中输入 `[[认知`
- **那么** 系统必须显示联想面板，并实时列出匹配“认知”的本地知识节点和摘录候选

#### 场景:关闭联想面板
- **当** 用户删除触发串、点击面板外区域或完成插入
- **那么** 系统必须关闭联想面板

### 需求:联想插入必须写入结构化链接文本
系统必须在用户选择联想项时，将链接以结构化文本形式写入本地内容，并在编辑区显示为 `[[标题]]` 的可读形式。

#### 场景:插入节点链接
- **当** 用户点击联想面板中的某个知识节点
- **那么** 系统必须在光标位置插入该节点对应的结构化链接文本，并保留该节点的本地标识用于后续解析

#### 场景:插入摘录链接
- **当** 用户点击联想面板中的某条摘录
- **那么** 系统必须在光标位置插入该摘录对应的结构化链接文本，并保留该摘录的本地标识用于后续解析

### 需求:展示态链接必须高亮并支持本地跳转或预览
系统必须在展示摘录想法和节点总结时解析结构化链接文本，并将其渲染为高亮、可点击的本地链接。

#### 场景:点击节点链接
- **当** 用户在节点详情页或摘录展示区点击一个解析后的节点链接
- **那么** 系统必须跳转到对应的节点详情页

#### 场景:点击摘录链接
- **当** 用户点击一个解析后的摘录链接
- **那么** 系统必须打开对应摘录的本地预览，或将界面定位到该摘录内容
