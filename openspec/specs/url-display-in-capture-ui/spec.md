# 规范：url-display-in-capture-ui

## 目的

在极速摘录弹窗（`CaptureDialogV2`）中展示来源相关的 UI 区域，根据标准解析与最近页面补齐的结果，提供来源标题、域名、补齐提示和可折叠链接输入等交互模式。

## 需求

### 需求:快摘弹窗必须展示来源标题

当系统已经解析或补齐出来源标题时，快摘弹窗必须在来源信息区优先展示来源标题，并与来源域名同时呈现。

#### 场景:展示来源标题与域名

- **当** 快摘初始化结果包含非空 `sourceTitle`
- **那么** 系统必须在摘录卡片下方的来源信息区展示来源标题
- **那么** 系统必须继续展示来源 URL 的域名或可读链接标识，且不得只显示完整裸链接

### 需求:快摘弹窗必须提示来源补齐状态

系统必须在来源信息由最近页面上下文缓存补齐时，以非阻塞方式提示用户当前来源来自辅助识别。

#### 场景:来源由最近页面上下文补齐

- **当** 当前弹窗中的标题或 URL 来自最近页面上下文缓存
- **那么** 系统必须展示一条非阻塞提示，说明来源已由最近页面自动补齐

#### 场景:来源缺失且可启用辅助识别

- **当** 当前来源应用属于受支持宿主，且弹窗仍缺失标题或 URL，同时辅助识别模式尚未开启
- **那么** 系统必须展示可忽略的引导入口，允许用户跳转开启辅助识别
- **那么** 系统不得强制用户先开启辅助识别后再保存摘录

### 需求：来源域名徽章（URL 提取成功）

当系统成功解析或补齐到有效来源信息时，必须在弹窗中展示只读的来源信息区；若已获得来源标题，则必须优先展示标题，并同时显示来源域名。

#### 场景：展示来源信息区

- **当** `initialUrl` 不为空字符串或 `sourceTitle` 不为空字符串
- **那么** 系统必须在书摘卡片下方展示来源信息区
- **那么** 若 `sourceTitle` 存在，来源信息区必须展示标题主文案与 URL 的 `host` 辅助文案
- **那么** 若 `sourceTitle` 不存在，来源信息区必须退化为只显示 URL 域名
- **那么** 来源信息区必须保持只读，不可直接编辑标题

#### 场景：来源信息区样式

- **当** 来源信息区渲染时
- **那么** 系统必须保持与现有快摘视觉一致的紧凑样式
- **那么** 标题、域名与补齐提示必须在半屏弹窗内清晰可读且不挤压正文输入主区域

### 需求：可折叠 URL 输入框（URL 提取失败）

当系统未能自动提取到 URL 时，必须提供折叠式的可选 URL 输入区域。

#### 场景：默认折叠状态

- **当** `initialUrl` 为空字符串，且最近页面上下文补齐也未生成有效 URL
- **那么** 显示 `CollapsibleUrlInput` 组件，初始状态为**折叠**（仅展示「添加来源链接 +」触发按钮）
- **那么** 折叠状态下保存按钮正常可用，用户可以不填 URL 直接提交

#### 场景：展开并输入

- **当** 用户点击「添加来源链接 +」触发按钮时
- **那么** 输入区域展开，显示 `OutlinedTextField`，用户可粘贴原文 URL
- **那么** 用户手动输入的链接必须覆盖自动补齐前的空值

#### 场景：空 URL 提交

- **当** 用户不展开或展开后输入框为空，点击「保存并继续心流」时
- **那么** 以空字符串作为 URL 字段值正常提交，后端已支持 URL 为空的摘录

### 需求：URL 字段数据流向

URL 与来源标题的最终值（标准解析、最近页面补齐或用户手动输入）必须正确传递至保存逻辑。

#### 场景：来源字段传递至保存函数

- **当** 用户点击保存时
- **那么** 保存逻辑中的 `url` 参数必须来自标准解析结果、最近页面补齐结果或用户手动输入结果之一
- **那么** 保存逻辑中的 `sourceTitle` 参数必须来自标准解析结果或最近页面补齐结果；若两者都缺失，则必须以空值保存
