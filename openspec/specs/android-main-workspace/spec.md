# 规范：android-main-workspace

## 目的
定义 Android 端本地主工作台的页面结构、导航状态和空状态引导，确保 `MainActivity` 中的工作台体验与 `doc/ui.tsx` 一致。

## 需求

### 需求:Android 主工作台必须严格遵循 ui.tsx 页面结构
系统必须在 `MainActivity` 中提供严格遵循 `doc/ui.tsx` 的主工作台，并按设计稿展示摘录流、知识树、标签、AI 提炼页、节点详情页、底部导航和悬浮 FAB；不得以功能相近但结构不同的布局替代。

#### 场景:展示摘录流首页
- **当** 用户从启动器打开应用且本地数据已经完成加载
- **那么** 系统必须默认显示摘录流页面，并按 `doc/ui.tsx` 的顺序展示搜索栏、筛选按钮、长按提示、摘录卡片列表、AI 推荐卡、右下 FAB 与底部导航

#### 场景:切换知识树与标签页
- **当** 用户点击底部导航中的“知识树”或“标签”
- **那么** 系统必须切换到对应页面，并保留 `doc/ui.tsx` 中的页标题、主操作按钮、列表层级与 FAB 位置

### 需求:主工作台必须支持 ui.tsx 定义的导航与多选状态
系统必须支持摘录流多选、AI 提炼页跳转、节点详情跳转以及返回栈行为，并在多选或子页面状态下正确切换底部区域。

#### 场景:进入多选模式
- **当** 用户在摘录流中长按某张摘录卡片
- **那么** 系统必须进入多选模式，显示已选数量与“取消 / AI提炼并归档”操作栏，并隐藏默认底部导航和 FAB

#### 场景:进入 AI 提炼页
- **当** 用户在多选模式下至少选中一条摘录并点击“AI提炼并归档”
- **那么** 系统必须进入 AI 提炼页，并保留所选摘录用于后续本地大纲生成

#### 场景:进入节点详情页
- **当** 用户在知识树页面点击某个节点标题
- **那么** 系统必须打开节点详情页，并提供返回到知识树页面的导航动作

### 需求:空库时必须提供引导和示例数据入口
当本地数据库中不存在工作台数据时，系统必须先展示空状态引导，并提供加载示例数据与开始记录的入口。

#### 场景:首次进入且本地库为空
- **当** 用户首次打开应用且 Room 中没有摘录、节点和标签数据
- **那么** 系统必须显示空状态说明，解释摘录流、知识树和 FAB 的作用，并提供“加载示例数据”和“开始记录”入口

#### 场景:加载示例数据
- **当** 用户在空状态下点击“加载示例数据”
- **那么** 系统必须写入与 `doc/ui.tsx` 对齐的本地示例数据，并立即展示可交互的主工作台页面

#### 场景:直接开始记录
- **当** 用户在空状态下点击“开始记录”
- **那么** 系统必须打开全局快速记录面板，而不得预先写入示例数据
