# 规范：settings-ui-redesign

## 目的
定义设置页面整体 UI 重设计的需求，包括页面布局结构、头像卡片、登录入口重构、外观设置区域和同步区域的样式统一，以及退出登录的位置。

## 需求

### 需求:设置页面必须按照新布局结构组织
系统必须将设置页面按以下顺序组织：头像区域（顶部）→ 外观设置区域 → 同步区域 → 退出登录（底部），各区域使用统一的 Material You 设计风格。

#### 场景:渲染完整设置页面
- **当** 用户进入设置页面
- **那么** 系统必须按顺序展示：头像区域、外观设置卡片、同步卡片
- **那么** 已登录时必须在页面最底部展示退出登录按钮
- **那么** 未登录时禁止展示退出登录按钮

### 需求:头像区域必须居顶显示用户信息
系统必须在设置页面顶部以居中列布局显示用户头像和信息，不使用 Card 包裹。

#### 场景:已登录状态的头像区域
- **当** 用户已登录
- **那么** 系统必须显示圆形用户头像（可点击更换）
- **那么** 系统必须在头像下方显示用户邮箱
- **那么** 系统禁止在头像区域显示退出登录按钮

#### 场景:未登录状态的头像区域
- **当** 用户未登录
- **那么** 系统必须显示默认占位头像
- **那么** 系统必须在头像下方显示引导文字（如"解锁完整体验"）
- **那么** 系统必须显示"登录"按钮，点击后导航到独立的登录/注册页面

### 需求:登录/注册必须作为独立全屏页面
系统必须将登录和注册表单从设置页面中分离，作为独立的全屏页面通过导航路由访问。

#### 场景:从设置页面导航到登录页
- **当** 未登录用户在设置页面点击"登录"按钮
- **那么** 系统必须导航到独立的登录/注册全屏页面

#### 场景:登录/注册成功后返回
- **当** 用户在登录/注册页面成功完成登录或注册
- **那么** 系统必须自动返回设置页面
- **那么** 设置页面必须立即更新为已登录状态

#### 场景:取消登录返回
- **当** 用户在登录/注册页面点击返回按钮
- **那么** 系统必须返回设置页面
- **那么** 系统必须保持未登录状态不变

### 需求:退出登录必须显示在设置页面最后一项
系统必须将退出登录按钮放置在设置页面最底部，与上方设置区域保持明显的间距分隔。

#### 场景:退出登录按钮位置
- **当** 已登录用户查看设置页面
- **那么** 退出登录按钮必须显示在所有设置区域的下方
- **那么** 退出登录按钮与上方最后一个设置区域之间必须有明显的间距（至少 24dp）
- **那么** 退出登录按钮必须居中显示

#### 场景:点击退出登录
- **当** 用户点击退出登录按钮
- **那么** 系统必须执行退出登录逻辑
- **那么** 设置页面必须立即切换为未登录状态
- **那么** 退出登录按钮必须消失

### 需求:外观设置区域和同步区域必须使用统一的卡片样式
系统必须使用 Material 3 的卡片组件（ElevatedCard 或 OutlinedCard）包裹外观设置和同步区域，保持视觉一致性。

#### 场景:渲染外观设置卡片
- **当** 设置页面展示外观设置区域
- **那么** 系统必须使用卡片组件包裹外观区域
- **那么** 卡片内必须包含"外观"标题和主题模式、主题颜色选择器

#### 场景:渲染同步卡片
- **当** 设置页面展示同步区域
- **那么** 系统必须使用与外观设置区域相同风格的卡片组件
- **那么** 卡片内必须包含同步状态信息和立即同步按钮
