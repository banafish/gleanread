# 规范：android-quick-capture-sheet

## 目的
定义 Android 主工作台中的快速记录 Bottom Sheet 行为，确保本地记录、推荐标签和 `[[` 联想体验符合设计稿。

## 需求

### 需求:主 Tab 的 FAB 必须打开与 ui.tsx 对齐的快速记录面板
系统必须在摘录流、知识树和标签页展示全局 FAB；点击后必须从底部滑出与 `doc/ui.tsx` 对齐的快速记录 Bottom Sheet，并自动聚焦输入框与弹出软键盘。

#### 场景:打开快速记录面板
- **当** 用户在任一主 Tab 点击右下角 FAB
- **那么** 系统必须显示半屏 Bottom Sheet、聚焦正文输入框、弹出软键盘，并保留底层页面作为变暗背景

#### 场景:关闭快速记录面板
- **当** 用户点击遮罩区域或向下拖动面板手柄
- **那么** 系统必须关闭 Bottom Sheet，且不得写入任何新摘录

### 需求:快速记录必须只写本地库并默认归档到 Inbox
系统必须将快速记录面板中的保存行为限制为本地写入，并在未显式选择其他归档位置时默认保存到 Inbox。

#### 场景:保存新摘录到 Inbox
- **当** 用户在快速记录面板中输入正文并点击发送
- **那么** 系统必须只把摘录写入本地 Room、默认令 `tree_node_id` 为空、把记录标记为待同步，并立即刷新摘录流

#### 场景:填写可选字段
- **当** 用户为新摘录填写 URL、标签或归档位置
- **那么** 系统必须把这些可选字段一并写入本地库，并保持正文之外的字段均为非必填

### 需求:快速记录面板必须支持标签推荐与 `[[` 联想能力
系统必须在快速记录面板中提供可点击的标签推荐，并允许用户在输入区通过 `[[` 触发本地双向链接联想。

#### 场景:显示标签推荐
- **当** 系统打开快速记录面板且本地库中存在高频标签
- **那么** 系统必须显示可点击的推荐标签胶囊，并允许用户快速勾选

#### 场景:在面板中触发双向链接联想
- **当** 用户在快速记录输入区输入 `[[`
- **那么** 系统必须按 `android-inline-link-composer` 的规则展示本地联想面板并完成链接插入
