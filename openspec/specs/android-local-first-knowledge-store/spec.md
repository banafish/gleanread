# 规范：android-local-first-knowledge-store

## 目的
定义 Android 端本地优先知识库的 Room 存储、查询派生和待同步标记规则，保证工作台在离线状态下仍由本地数据完整驱动。

## 需求

### 需求:Room 本地知识库必须落地四表与同步字段
系统必须在 Android 端使用 Room 持久化 `knowledge_tree_node`、`tags`、`excerpts`、`excerpt_tags` 四张表，并保留 `id`、`user_id`、`is_deleted`、`sync_status` 等离线优先字段。

#### 场景:初始化本地数据库
- **当** 应用首次创建本地数据库
- **那么** 系统必须创建上述四张表及其主键和索引，并使新增记录能够携带待同步状态

#### 场景:保存新的未归档摘录
- **当** 用户通过主工作台或快速记录面板新建一条摘录且未选择知识树节点
- **那么** 系统必须把记录写入 `excerpts` 表、令 `tree_node_id` 为空、把 `sync_status` 标记为待同步，并为关联标签写入 `excerpt_tags` 关系与本地热度更新

### 需求:主工作台查询必须完全由本地库派生
主工作台的 Inbox、知识树、标签分组、节点详情和 Backlinks 必须只从 Room 查询结果派生，不得依赖现有后端接口返回。

#### 场景:离线浏览工作台
- **当** 设备处于无网络状态且本地库中已经存在数据
- **那么** 系统必须仍能展示摘录流、知识树、标签页和节点详情，并允许用户继续浏览本地内容

#### 场景:构建知识树与标签页
- **当** 本地库中存在树节点、摘录和标签关系
- **那么** 系统必须从 `parent_id` 构造知识树层级、从 `tree_node_id IS NULL` 派生 Inbox、并按本地标签名称与热度生成标签页分组

### 需求:本地变更必须维护待同步标记
任何由主工作台发起的新增、编辑和挂载操作都必须只写本地库，并维护受影响记录的待同步状态。

#### 场景:挂载摘录到知识树节点
- **当** 用户在 AI 提炼页把未归档摘录挂载到某个节点
- **那么** 系统必须只更新本地表中的节点和摘录记录，并把受影响的数据标记为待同步，而不得调用现有后端接口

### 需求:示例数据导入必须复用正式表结构
系统必须通过与正式数据相同的 Room 表结构写入示例数据，并使示例数据可以直接驱动主工作台、AI 提炼页和节点图谱。

#### 场景:导入示例数据
- **当** 用户选择加载示例数据
- **那么** 系统必须写入与 `doc/ui.tsx` 一致的摘录、标签、节点及关联关系，并保证它们可被摘录流、知识树、标签页、节点详情和局部图谱直接查询
