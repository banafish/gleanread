# 规范：storage-management

## 目的
定义多账号本地数据库文件的空间管理与清理需求。

## 需求

### 需求:历史数据库文件清理机制
系统必须提供管理多个 `local_data_*.db` 文件的机制，避免长期的多账号切换导致存储空间无限膨胀。

#### 场景:设置页手动清理
- **当** 用户点击设置页面的“清除本地缓存”按钮时
- **那么** 系统必须删除除了当前正在使用的数据库文件之外的所有其他 `local_data_*.db` 和 `guest.db` 文件

#### 场景:应用启动自动清理超期库
- **当** App 每次冷启动或初始化存储服务时
- **那么** 系统必须检测所有非当前的 `.db` 文件，并自动删除最后修改时间超过 30 天的过期文件
