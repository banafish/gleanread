# 规范：sync-queue-guard

## 目的
定义退出登录前的同步队列检查与离线数据保护需求。

## 需求

### 需求:离线数据队列与登出安全拦截
系统必须在用户尝试退出登录时，检查是否有未同步至云端的本地数据修改，并给出适当警告，防止数据在离线设备上孤立或丢失。

#### 场景:断网修改后直接登出拦截
- **当** 用户在无网络连接状态下进行了数据修改，尝试点击“退出登录”时
- **那么** 系统必须检测到同步队列不为空，并弹出弹窗警告用户“有未同步数据，退出后只能在本机查看，是否继续退出？”

#### 场景:正常同步完成后登出
- **当** 用户的本地修改已全部推送至 Supabase 云端，同步队列为空，此时尝试退出登录时
- **那么** 系统禁止弹出警告，直接执行退出流程
