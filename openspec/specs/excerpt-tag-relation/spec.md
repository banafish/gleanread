# 规范：excerpt-tag-relation

## 目的
定义摘录与标签之间的本地多对多关联持久化与查询规则，确保摘录入库、后续编辑和按标签筛选都能复用同一份关联数据。

## 需求

### 需求：摘录与标签的多对多关联持久化

系统必须在摘录入库和后续标签编辑时维护 `excerpt_tags` 关联表，并提供按标签反向检索摘录的能力，以支撑 AI 合成的选料前序流程。

#### 场景：带标签摘录入库，关联记录正确写入

- **当** 移动端提交一条附带 `["架构", "后端"]` 标签的摘录
- **那么** 系统在 `excerpts` 表新增摘录记录，在 `tags` 表累加 `#架构` 和 `#后端` 的 `heatWeight`，并在 `excerpt_tags` 表写入两条关联记录 `(excerpt_id, tag_id)`

#### 场景：编辑已有摘录标签

- **当** 用户在摘录详情页调整某条摘录的标签集合并保存
- **那么** 系统必须使该摘录在 `excerpt_tags` 表中的关联记录与最新标签集合保持一致
- **那么** 新增的标签关联必须写入待同步关系记录，移除的标签关联必须以本地删除或等价方式标记为待同步删除

#### 场景：按标签筛选 Inbox 摘录

- **当** 前端或用户在 AI 合成前，通过标签筛选某标签下的待处理摘录
- **那么** 系统必须通过 `excerpt_tags` 关联表返回该标签下所有 `treeNodeId IS NULL` 的摘录集合
