# 规范：android-knowledge-tree-browser

## 目的
定义 Android 端知识树首页与分支页的浏览结构、节点跳转规则和页面级操作，确保知识树浏览体验与 `doc/知识树页面设计260418.md` 一致。

## 需求

### 需求:知识树首页必须提供浅层树预览与空状态
系统必须按照 `doc/知识树页面设计260418.md` 将知识树首页呈现为浅层树预览页：以一级节点卡片承载根节点，并在卡片内最多展示二级与三级节点；当本地不存在任何知识树节点时，系统必须展示知识树空状态与新增根节点入口。

#### 场景:首页展示根节点卡片
- **当** 用户打开知识树首页且本地存在一级知识树节点
- **那么** 系统必须以卡片列表展示所有一级节点
- **那么** 每张卡片必须显示节点名称、条数、展开状态与二级节点预览

#### 场景:首页展示三级浅预览
- **当** 用户展开首页某个一级节点卡片且该根节点存在二级或三级节点
- **那么** 系统必须在同一卡片内继续展示二级节点及其可见的三级节点
- **那么** 首页不得继续直接展开四级及以下节点

#### 场景:知识树为空
- **当** 用户打开知识树首页且本地不存在任何知识树节点
- **那么** 系统必须显示树形空状态说明文案与“新增节点”入口
- **那么** 空状态下仍必须保留知识树首页顶部栏

### 需求:知识树分支页必须提供面包屑局部子树浏览
系统必须提供基于当前节点的局部子树页，并显示从知识树首页到当前节点的面包屑路径；分支页必须从当前节点开始继续展示后三层局部子树，只有超过当前三级局部预览可见深度的节点才允许通过“进入 >”或等价入口继续下钻。

#### 场景:进入分支页显示路径和局部子树
- **当** 用户从知识树首页或其他分支页进入某个节点的分支页
- **那么** 系统必须在页面顶部显示返回入口、当前节点标题与面包屑路径
- **那么** 页面内容必须展示当前节点的直接子节点以及这些子节点可见的后两层预览

#### 场景:分支页继续深入
- **当** 分支页中的某个可见节点下仍存在超出当前页三级局部预览可见深度的后代节点
- **那么** 系统必须为该节点显示“进入 >”或等价的继续深入入口
- **那么** 用户点击该入口后系统必须打开更深一层的知识树分支页

### 需求:知识树浏览必须根据层级规则决定展开与跳转
系统必须根据当前页面的可见深度决定节点行为：首页一级节点卡片主体必须进入对应分支页，展开控件只负责切换预览；首页或分支页中的其他可见节点在无隐藏更深层级时必须进入节点详情页，在仍有隐藏更深层级时必须进入知识树分支页。

#### 场景:点击首页一级节点卡片主体
- **当** 用户点击知识树首页某个一级节点卡片主体
- **那么** 系统必须打开该一级节点对应的知识树分支页
- **那么** 系统不得把该点击误判为展开或收起操作

#### 场景:点击可直接查看的节点
- **当** 用户点击知识树首页或分支页中一个没有隐藏更深层级的可见节点标题
- **那么** 系统必须打开该节点的详情页

#### 场景:点击需要继续浏览的节点
- **当** 用户点击知识树首页或分支页中一个存在隐藏更深层级后代的可见节点
- **那么** 系统必须打开该节点的知识树分支页，而不是直接跳到详情页

#### 场景:点击展开控件
- **当** 用户点击知识树首页一级卡片或分支页可展开节点的展开控件
- **那么** 系统必须仅切换当前节点在本页的展开或收起状态
- **那么** 系统不得在切换展开状态时触发页面跳转

### 需求:知识树页面必须提供搜索入口与页面级更多操作
系统必须在知识树首页与分支页顶部提供搜索入口和页面级更多菜单；搜索内容必须展示最近搜索、节点结果和内容结果，页面级更多菜单必须至少支持“展开全部”和“收起全部”。

#### 场景:打开知识树搜索内容
- **当** 用户在知识树首页或分支页点击搜索入口
- **那么** 系统必须显示知识树搜索内容，并基于本地节点与摘录展示最近搜索、节点结果和内容结果
- **那么** 搜索结果中的节点项必须显示其所在路径信息

#### 场景:展开全部与收起全部
- **当** 用户在知识树首页或分支页点击顶部更多菜单中的“展开全部”或“收起全部”
- **那么** 系统必须只更新当前知识树页面的展开状态
- **那么** 更新后的展开状态必须立即反映到当前页面的节点预览
