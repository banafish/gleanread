## 新增需求

### 需求:多用户本地数据库隔离与切换
系统必须使用独立的文件级别隔离不同账号和访客状态的数据。每个用户必须有专属的 `.db` 文件。

#### 场景:访客状态初始化
- **当** 用户未登录且无历史登录信息启动 App 时
- **那么** 系统必须初始化并打开名为 `guest.db` 的数据库实例用于读写

#### 场景:新用户首次登录
- **当** 用户A 首次在设备上登录成功获取到 `user_id` 时
- **那么** 系统必须创建并打开一个名为 `local_data_{user_id}.db` 的空数据库，并从 Supabase 拉取云端最新数据

#### 场景:老用户切换登录 (秒开体验)
- **当** 用户A 在设备上登录过，退出后重新登录时
- **那么** 系统必须立即打开本地已有的 `local_data_{user_id}.db` 文件并直接用于 UI 渲染，后台静默拉取云端增量数据
