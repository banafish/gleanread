## 为什么

当前 Android 知识树页仍是单文件递归展开列表，只支持新增根节点，无法承载 `doc/知识树页面设计260418.md` 中要求的首页浅层预览、面包屑局部子树浏览、统一节点菜单、搜索入口和完整空状态。现在需要把知识树改造成分段浏览与节点管理并存的 Material 3 页面结构，同时保留现有“点击节点进入详情页”的核心导航能力。

## 变更内容

- 将知识树页面从单一递归树列表重构为“知识树首页 + 面包屑分支页”的双页面结构：首页展示一级节点卡片及二、三级浅预览，四级及以下通过局部子树页继续浏览。
- 新增知识树专属组件与状态模型，包括顶部栏、首页与分支页 FAB、面包屑路径、节点更多菜单、搜索内容、空状态，以及新增/重命名/删除节点对话框。
- 按建议目录将知识树代码拆分到 `feature/knowledge_tree/` 下的 route、screen、component、model，替换现有 `TreeScreen.kt` 的单文件实现。
- 保留现有节点详情导航语义：可直接查看详情的节点继续进入 `node/{nodeId}`；需要继续下钻浏览的节点进入知识树分支页，不改变既有详情页能力和返回栈预期。
- 补齐首页与分支页的展开/收起、统一节点操作和局部子树预览规则，使页面行为严格对齐设计文档。

## 功能 (Capabilities)

### 新增功能

- `android-knowledge-tree-browser`: 定义知识树首页、面包屑分支页、搜索内容、空状态、浅层预览和更深层跳转规则，确保知识树浏览结构严格对齐 `知识树页面设计260418.md`。
- `android-knowledge-tree-node-actions`: 定义所有层级节点统一的新增子节点、重命名、删除菜单与对话框，以及首页新增根节点、分支页新增子节点的交互行为。

### 修改功能

- `android-main-workspace`: 更新主工作台中知识树 Tab 的路由、页面结构和导航要求，使其接入新的知识树首页/分支页，同时继续支持从知识树点击节点进入详情页并正确返回。

## 影响

- Android UI 代码将新增 `feature/knowledge_tree/` 目录，并调整 `WorkspaceScreen` 中知识树相关导航入口。
- `WorkspaceViewModel`、`WorkspaceRepository`、Room DAO/实体映射可能需要补充节点新增、重命名、删除子树和分支浏览所需的数据操作。
- 现有 `TreeNodeUiModel` 与 `WorkspaceSnapshot` 很可能需要增加适配新页面的数据结构或转换层。
- 不涉及新的后端 API；主要影响 Android 本地状态管理、导航与知识树展示逻辑。
