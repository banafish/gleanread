## 新增需求

### 需求:所有知识树节点必须提供统一更多菜单
系统必须在知识树首页一级节点卡片、二三级节点行和分支页节点行提供统一的更多菜单，且菜单项必须至少包含“新增子节点”“重命名”“删除”。

#### 场景:打开一级节点更多菜单
- **当** 用户在知识树首页点击某个一级节点卡片右侧的更多按钮
- **那么** 系统必须显示包含“新增子节点”“重命名”“删除”的菜单

#### 场景:打开二级或分支节点更多菜单
- **当** 用户在知识树首页二三级节点行或分支页节点行点击更多按钮
- **那么** 系统必须显示与一级节点一致的菜单项
- **那么** 系统不得因节点层级不同而隐藏上述三项基础操作

### 需求:知识树必须支持新增根节点与新增子节点
系统必须支持从知识树首页新增根节点，也必须支持从知识树分支页或节点更多菜单新增子节点；新增节点必须通过对话框输入名称，并在保存后立即反映到当前知识树页面。

#### 场景:首页新增根节点
- **当** 用户在知识树首页点击新增根节点 FAB 或空状态中的“新增节点”入口
- **那么** 系统必须打开新增节点对话框
- **那么** 用户保存后系统必须创建新的一级节点并立即刷新知识树首页

#### 场景:分支页新增当前节点子节点
- **当** 用户在某个知识树分支页点击“新增子节点” FAB
- **那么** 系统必须打开新增子节点对话框，并默认将当前分支节点作为父节点
- **那么** 用户保存后系统必须把新节点显示在当前分支页

#### 场景:从节点菜单新增任意子节点
- **当** 用户在任意知识树节点的更多菜单中点击“新增子节点”
- **那么** 系统必须打开新增子节点对话框，并显示所选节点作为父节点
- **那么** 用户保存后系统必须把新节点追加到该父节点下

### 需求:知识树必须支持重命名节点
系统必须允许用户通过重命名对话框修改任意知识树节点名称，并在保存后立即更新当前知识树页面和后续导航中显示的节点标题。

#### 场景:打开重命名对话框
- **当** 用户在任意知识树节点的更多菜单中点击“重命名”
- **那么** 系统必须打开重命名对话框，并预填当前节点名称

#### 场景:保存重命名
- **当** 用户在重命名对话框中输入新名称并确认保存
- **那么** 系统必须更新该节点名称
- **那么** 当前知识树页面、面包屑路径与后续节点跳转标题必须立即显示新名称

### 需求:知识树必须在删除前确认并移除整棵子树
系统必须在删除节点前展示确认对话框；若目标节点存在后代节点，确认文案必须明确提示子树数量与整棵子树都会被移除。用户确认后，系统必须从知识树中移除该节点及其所有后代节点。

#### 场景:删除普通节点
- **当** 用户对一个没有子节点的知识树节点执行删除
- **那么** 系统必须显示“删除后不可恢复”的确认提示
- **那么** 用户确认后系统必须从当前知识树页面移除该节点

#### 场景:删除存在子树的节点
- **当** 用户对一个存在后代节点的知识树节点执行删除
- **那么** 系统必须在确认对话框中显示该节点下仍有子节点以及“整棵子树都会被移除”的提示
- **那么** 用户确认后系统必须从知识树中移除该节点及所有后代节点
