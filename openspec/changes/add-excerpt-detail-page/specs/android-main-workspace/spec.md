## MODIFIED Requirements

### 需求:主工作台必须支持 ui.tsx 定义的导航与多选状态
系统必须支持摘录流多选、AI 提炼页跳转、知识树分支页跳转、节点详情跳转、摘录详情跳转以及返回栈行为，并在多选或子页面状态下正确切换底部区域。

#### 场景:进入多选模式
- **当** 用户在摘录流中长按某张摘录卡片
- **那么** 系统必须进入多选模式，显示已选数量与“取消 / AI提炼并归档”操作栏，并隐藏默认底部导航和 FAB

#### 场景:进入 AI 提炼页
- **当** 用户在多选模式下至少选中一条摘录并点击“AI提炼并归档”
- **那么** 系统必须进入 AI 提炼页，并保留所选摘录用于后续本地大纲生成

#### 场景:进入知识树分支页
- **当** 用户在知识树首页或知识树分支页点击一个仍有隐藏更深层级的节点
- **那么** 系统必须打开对应的知识树分支页
- **那么** 该子页面必须隐藏默认底部导航和全局 FAB，并提供返回到上一层知识树页面的导航动作

#### 场景:进入节点详情页
- **当** 用户在知识树首页或知识树分支页点击一个可直接查看详情的节点标题
- **那么** 系统必须打开节点详情页，并提供返回到触发来源知识树页面的导航动作

#### 场景:从摘录流进入摘录详情页
- **当** 用户在摘录流中点击某条摘录卡片
- **那么** 系统必须打开对应摘录的详情页
- **那么** 用户返回时系统必须回到摘录流，并保留原有的列表上下文

#### 场景:从节点详情进入摘录详情页
- **当** 用户在节点详情页点击挂载摘录、Backlinks 中的摘录条目，或点击解析后的摘录链接
- **那么** 系统必须打开对应摘录的详情页
- **那么** 用户返回时系统必须回到触发该跳转的节点详情页

#### 场景:从局部图谱进入摘录详情页
- **当** 用户在局部关系图中点击某个摘录节点
- **那么** 系统必须打开对应摘录的详情页
- **那么** 用户返回时系统必须回到原来的局部关系图视图
