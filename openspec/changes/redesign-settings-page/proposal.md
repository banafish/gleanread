## 为什么

当前设置页面采用简单的 Card + 表单布局，缺少用户个性化体验：无头像展示、无外观（浅色/深色/跟随系统）和主题颜色选择。页面整体设计较为朴素，没有视觉层次感。参照截图中的设计风格（头像卡片居顶、外观模式切换区、主题色选择器），需要对设置页面进行完整的 UI 重设计，使其更具 Material You 设计感和可交互性。

## 变更内容

1. **用户头像区域**：在设置页面顶部新增用户头像展示区域。已登录时显示头像（支持更换）+ 邮箱；未登录时显示默认占位头像 + "登录" 引导按钮。
2. **登录/注册流程 UI 重构**：将登录表单从设置页面主体中分离为独立的登录/注册页面（全屏或 Bottom Sheet），支持邮箱密码登录和邮箱密码注册（新增），设置页面仅保留头像卡片和"登录"入口按钮。登录或注册成功后头像区域切换为已登录状态，显示邮箱信息。
3. **外观设置**：新增外观设置区域，包含：
   - **主题模式切换**：浅色 / 深色 / 跟随系统三选一，参照截图的可选卡片样式。
   - **主题颜色选择**：动态（Material You）、海洋、紫色、森林、樱花、琥珀、石墨等预设主题色选择器，参照截图的圆形色块带勾选样式。
4. **主动同步**：保留现有同步功能，调整 UI 样式与新设计统一。
5. **退出登录**：已登录状态下，退出登录按钮显示在设置页面最后一项，与上方其他设置区域保持明显的间距分隔，不放在头像区域内。

## 功能 (Capabilities)

### 新增功能
- `settings-appearance`: 外观设置功能，包括主题模式（浅色/深色/跟随系统）切换和主题颜色（动态/预设色）选择，需要新增偏好持久化和主题切换机制。
- `settings-avatar`: 用户头像管理功能，包括头像展示、更换头像（从相册选取或拍照），头像上传并存储在 Supabase Storage 服务上。
- `settings-ui-redesign`: 设置页面整体 UI 重设计，包括头像卡片布局、登录入口重构（分离为独立页面）、外观设置区域、同步区域样式统一。

### 修改功能
- `android-supabase-auth`: 登录/注册的 UI 入口从设置页面内联表单改为独立页面，新增邮箱密码注册功能（调用 Supabase signUp API），同时调整 UI 层面的导航路由和状态传递。
- `android-supabase-cloud-sync`: 同步区域 UI 样式调整以匹配新设计，不改变同步逻辑本身。

## 影响

- **UI 层**：`feature/settings/` 下所有文件需要大幅重构；`core/ui/theme/Theme.kt` 需要支持外部传入主题模式和主题颜色参数。
- **新增偏好存储**：需要新增 DataStore 或 SharedPreferences 存储用户外观偏好（主题模式、主题颜色）。
- **Supabase Storage**：头像文件上传到 Supabase Storage，需要配置 Storage bucket 和上传/下载逻辑。
- **导航**：需要新增登录/注册页面的导航路由（NavGraph 变更）。
- **主题系统**：`GleanReadTheme` composable 当前硬编码 `dynamicColor = true`，需要改为支持用户选择的主题模式和颜色方案。
- **依赖**：可能需要引入 Coil 或类似图片加载库用于头像显示（需确认项目是否已有）。
