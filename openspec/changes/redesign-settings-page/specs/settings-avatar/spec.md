# 规范：settings-avatar

## 目的
定义用户头像管理功能的需求，包括头像展示、更换和云端存储。

## 新增需求

### 需求:设置页面必须展示用户头像
系统必须在设置页面顶部以圆形头像展示当前用户的头像图片，根据登录状态显示不同内容。

#### 场景:已登录且有头像
- **当** 用户已登录且 Supabase Storage 中存在该用户的头像
- **那么** 系统必须加载并显示用户的圆形头像

#### 场景:已登录但无头像
- **当** 用户已登录但 Supabase Storage 中不存在该用户的头像
- **那么** 系统必须显示默认的占位头像图标

#### 场景:未登录
- **当** 用户未登录
- **那么** 系统必须显示默认的占位头像图标

### 需求:已登录用户必须能够更换头像
系统必须允许已登录用户通过点击头像触发更换操作，支持从相册选取或拍照获取图片。

#### 场景:点击头像触发更换
- **当** 已登录用户点击头像区域
- **那么** 系统必须弹出选择方式（从相册选取或拍照）

#### 场景:从相册选取图片
- **当** 用户选择从相册获取图片并成功选定一张图片
- **那么** 系统必须将图片压缩/缩放到合理尺寸
- **那么** 系统必须将图片上传到 Supabase Storage
- **那么** 系统必须在上传成功后更新设置页面显示的头像

#### 场景:拍照获取图片
- **当** 用户选择拍照并成功拍摄一张照片
- **那么** 系统必须将照片压缩/缩放到合理尺寸
- **那么** 系统必须将照片上传到 Supabase Storage
- **那么** 系统必须在上传成功后更新设置页面显示的头像

#### 场景:用户取消选择
- **当** 用户在选择相册图片或拍照过程中取消操作
- **那么** 系统必须保留当前头像不变
- **那么** 系统禁止显示错误提示

### 需求:头像必须存储在 Supabase Storage
系统必须将用户头像上传到 Supabase Storage 的 `avatars` bucket，以 `{userId}.jpg` 为文件路径。

#### 场景:上传头像成功
- **当** 系统将头像图片上传到 Supabase Storage
- **那么** 系统必须使用 `avatars/{userId}.jpg` 作为存储路径
- **那么** 若该路径已有旧头像，系统必须覆盖旧文件
- **那么** 系统必须将生成的 Public URL 缓存到本地

#### 场景:上传头像失败
- **当** 头像上传到 Supabase Storage 失败（网络异常等）
- **那么** 系统必须保留当前头像不变
- **那么** 系统必须显示上传失败的提示信息

#### 场景:头像 URL 本地缓存
- **当** 系统成功获取或上传头像后
- **那么** 系统必须将头像 Public URL 缓存到本地 DataStore
- **那么** 下次应用启动时必须优先使用缓存 URL 加载头像，避免重复请求

### 需求:未登录用户禁止更换头像
系统禁止未登录用户触发头像更换操作。

#### 场景:未登录时点击头像
- **当** 未登录用户点击头像区域
- **那么** 系统禁止弹出图片选择
- **那么** 系统必须引导用户前往登录页面（或无响应）
