# 增量规范：android-supabase-auth

## 新增需求

### 需求:系统必须支持邮箱密码注册
系统必须在 Android App 中提供 Supabase 邮箱密码注册能力，注册成功后根据 Supabase 配置决定是否需要邮箱验证。

#### 场景:注册成功且无需邮箱验证
- **当** 用户使用邮箱和密码完成注册且 Supabase 未开启邮箱验证
- **那么** 系统必须自动完成登录并持久化会话
- **那么** 系统必须返回设置页面并显示已登录状态

#### 场景:注册成功但需要邮箱验证
- **当** 用户使用邮箱和密码完成注册且 Supabase 开启了邮箱验证
- **那么** 系统必须提示用户查收验证邮件
- **那么** 系统禁止在邮箱验证完成前允许登录

#### 场景:注册失败（邮箱已存在）
- **当** 用户使用已注册的邮箱尝试注册
- **那么** 系统必须显示邮箱已注册的错误提示
- **那么** 系统必须保持当前注册表单状态不变

#### 场景:注册失败（密码不符合要求）
- **当** 用户输入的密码不符合 Supabase 密码策略
- **那么** 系统必须显示密码要求的错误提示

### 需求:注册页面必须验证确认密码
系统必须在注册模式下要求用户输入两次密码，且两次密码必须一致方可提交注册。

#### 场景:密码不一致
- **当** 用户输入的密码与确认密码不一致
- **那么** 系统必须在用户点击注册按钮时显示密码不一致的错误提示
- **那么** 系统禁止发送注册请求

#### 场景:密码一致
- **当** 用户输入的密码与确认密码一致
- **那么** 系统必须允许发送注册请求

## 修改需求

### 需求:Android 端必须支持 Supabase 登录会话
系统必须在 Android App 中提供 Supabase 邮箱密码登录能力，并在登录成功后持久化当前会话，使后续云端同步可以使用 Supabase 用户 ID 作为数据归属标识。

#### 场景:未登录时继续本地使用
- **当** 用户尚未登录 Supabase
- **那么** 系统必须允许用户继续使用本地工作台读写数据
- **那么** 系统必须禁止执行云端同步上传或拉取

#### 场景:登录成功后保存会话
- **当** 用户使用邮箱和密码完成 Supabase 登录
- **那么** 系统必须持久化登录会话
- **那么** 系统必须能够读取当前 Supabase 用户 ID 供同步层写入 `user_id`

#### 场景:登录失败
- **当** 用户输入的邮箱或密码无法完成 Supabase 登录
- **那么** 系统必须保留当前未登录或原会话状态
- **那么** 系统必须显示登录失败原因
- **那么** 系统禁止执行云端同步上传或拉取

#### 场景:会话失效
- **当** 已登录会话失效或刷新失败
- **那么** 系统必须停止云端同步
- **那么** 系统必须保留本地数据，并提示用户重新登录后再继续同步

#### 场景:登录/注册入口从独立页面访问
- **当** 未登录用户需要登录或注册
- **那么** 系统必须通过独立的全屏登录/注册页面提供入口
- **那么** 系统禁止在设置页面内嵌登录表单
- **那么** 登录/注册页面必须支持在登录模式和注册模式之间切换
