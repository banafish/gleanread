# 增量规范：android-supabase-cloud-sync

## 修改需求

### 需求:云端变更必须增量拉取到 Room
系统必须在登录后通过增量拉取把当前用户的 Supabase 数据写入本地 Room，并保持 UI 继续从 Room 派生。

#### 场景:应用启动后拉取云端数据
- **当** 用户已登录且应用启动
- **那么** 系统必须从 Supabase 拉取当前用户在上次拉取后变化的核心数据
- **那么** 系统必须在 Room transaction 中写入这些变化

#### 场景:拉取软删除记录
- **当** 云端返回 `is_deleted = true` 的记录
- **那么** 系统必须在本地保留该软删除标记
- **那么** 系统后续 UI 查询必须继续过滤已软删除数据

#### 场景:Realtime 通知触发拉取
- **当** 前台应用收到 Supabase Realtime 变更通知
- **那么** 系统必须触发对应数据范围的增量拉取
- **那么** 系统禁止只依赖 Realtime 通知本身作为最终数据源

#### 场景:设置页手动立即同步
- **当** 已登录用户在设置页触发立即同步
- **那么** 系统必须执行一轮上传本地待同步记录和拉取云端增量数据
- **那么** 系统禁止在主工作台页面暴露立即同步入口

#### 场景:同步区域 UI 样式
- **当** 设置页面展示同步区域
- **那么** 同步区域必须使用与外观设置区域一致的卡片风格
- **那么** 卡片内必须展示同步启用状态、上次同步时间、失败/冲突计数
- **那么** 卡片内必须包含"立即同步"按钮
