## 新增需求

### 需求:最近页面上下文缓存
系统必须在本地维护最近一条页面上下文快照，用于在外部快摘入口缺失来源字段时补齐标题和 URL。

#### 场景:缓存最近识别到的页面
- **当** 用户已开启辅助识别模式，且系统在受支持宿主中识别到有效的页面标题或 URL
- **那么** 系统必须把 `sourcePackage`、`sourceTitle`、`sourceUrl`、`capturedAt`、`captureSource` 和 `confidence` 写入最近页面上下文缓存

#### 场景:新快照覆盖旧快照
- **当** 系统识别到更新的页面上下文快照
- **那么** 系统必须只保留最近一条快照，并以新的页面上下文覆盖旧值

### 需求:最近页面上下文缓存必须受有效期约束
系统必须对最近页面上下文缓存应用严格有效期，禁止把过期快照用于新的快摘补齐。

#### 场景:读取未过期快照
- **当** 消费方读取最近页面上下文缓存，且当前时间仍处于有效期内
- **那么** 系统必须返回该快照用于后续来源补齐

#### 场景:读取过期快照
- **当** 消费方读取最近页面上下文缓存，且快照已超过有效期
- **那么** 系统必须把该快照视为不可用，并禁止用其补齐新的摘录来源

### 需求:辅助识别模式必须是显式开启的可选能力
系统必须仅在用户显式开启辅助识别模式后采集页面上下文，禁止在未授权状态下运行页面观察与缓存写入。

#### 场景:用户未开启辅助识别
- **当** 用户尚未在系统无障碍设置中开启 GleanRead 的辅助识别服务
- **那么** 系统必须停止页面上下文采集，并保持标准分享与手动补链路径可用

#### 场景:辅助识别服务识别受支持宿主
- **当** 用户已开启辅助识别模式，且当前前台应用属于受支持宿主
- **那么** 系统必须仅在识别出有效标题或 URL 时写入最近页面上下文缓存
