# 增量规范：node-reorder-interaction

## 新增需求

### 需求:知识树页面必须支持长按整张卡片拖拽同级排序
系统必须在知识树首页与分支页支持长按整张节点卡片进入拖拽模式，用户可在当前屏幕可见的同级节点间拖拽排序。拖拽仅支持同级排序移动，不支持跨层级移动，也不支持单独拖拽卡片内部的子节点。拖拽范围限于当前页面可见的同级节点。

#### 场景:长按整张卡片进入拖拽模式
- **当** 用户在知识树首页或分支页长按某张节点卡片（整张卡片，不可单独拖拽卡片内子节点）
- **那么** 系统必须触发触觉反馈（HapticFeedback）
- **那么** 被拖拽节点必须在视觉上抬起（elevation 提升、缩放反馈）
- **那么** 原位置必须显示半透明占位符

#### 场景:拖拽过程中其他节点视觉位移
- **当** 用户在拖拽模式下移动手指
- **那么** 被拖拽节点经过的位置，其他同级节点必须视觉上相应移动（被挤上来或挤下去）
- **那么** 其他节点的视觉位移必须直观地显示被拖拽节点被移动到了哪两个节点之间
- **那么** 系统必须通过其他节点的视觉位移来计算被拖拽节点的目标插入位置

#### 场景:拖拽过程中禁用其他交互
- **当** 节点处于拖拽模式
- **那么** 系统必须禁用节点的展开/收起操作
- **那么** 系统必须禁用节点的点击跳转操作
- **那么** 系统必须禁用节点的更多菜单操作

#### 场景:禁止单独拖拽卡片内子节点
- **当** 用户尝试长按卡片内部的子节点进行拖拽
- **那么** 系统禁止对卡片内子节点单独触发拖拽
- **那么** 长按整张卡片才是进入拖拽模式的唯一方式

### 需求:拖拽仅支持同级排序
系统必须允许用户通过拖拽在同一父节点的子节点列表内调整节点顺序，不支持跨层级移动。

#### 场景:同级节点向上拖拽
- **当** 用户将节点拖拽到同级列表中更高的位置
- **那么** 拖拽过程中，被拖拽节点经过的位置，上方的节点必须视觉上向下移动（让出位置）
- **那么** 拖拽结束后系统必须根据视觉位移确定的插入位置更新该节点的 `sort_order`，使其位于目标位置的前后兄弟节点之间
- **那么** 节点列表必须立即反映新的排列顺序

#### 场景:同级节点向下拖拽
- **当** 用户将节点拖拽到同级列表中更低的位置
- **那么** 拖拽过程中，被拖拽节点经过的位置，下方的节点必须视觉上向上移动（被挤上来）
- **那么** 拖拽结束后系统必须根据视觉位移确定的插入位置更新该节点的 `sort_order`，使其位于目标位置的前后兄弟节点之间
- **那么** 节点列表必须立即反映新的排列顺序

#### 场景:禁止跨层级拖拽移动
- **当** 用户尝试将节点拖拽到不同层级的位置
- **那么** 系统禁止执行跨层级移动
- **那么** 跨层级移动必须通过现有的 `MoveNodeBottomSheet` 操作完成

### 需求:拖拽取消必须恢复原始状态
系统必须支持用户在拖拽过程中取消操作，取消后节点必须恢复到拖拽前的位置和状态。

#### 场景:拖拽过程中松手未到达有效目标
- **当** 用户在拖拽过程中松手但指针不在任何有效 drop 目标区域
- **那么** 系统必须取消拖拽操作
- **那么** 被拖拽节点必须动画回到原始位置
- **那么** 其他节点的视觉位移必须动画恢复到原始位置
- **那么** 节点的 `sort_order` 禁止发生任何变更
