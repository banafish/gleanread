# 增量规范：node-sort-order

## 新增需求

### 需求:知识树节点必须具备稀疏排序字段
系统必须为每个知识树节点维护一个 `sort_order` 排序字段（Long 类型），用于决定同一父节点下子节点的显示顺序。排序字段必须使用稀疏策略（初始间隔 65536），使得在两个节点之间插入新节点时仅需计算中间值，无需批量更新所有兄弟节点。

#### 场景:数据库包含排序字段
- **当** 应用数据库升级到版本 3
- **那么** `knowledge_tree_node` 表必须包含 `sort_order INTEGER NOT NULL DEFAULT 0` 列
- **那么** 所有已有节点的 `sort_order` 必须为默认值 0

#### 场景:排序字段通过同步获取初始值
- **当** 应用完成数据库迁移后首次执行云同步
- **那么** 系统必须从 Supabase 拉取每个节点的 `sort_order` 值覆盖本地默认值 0
- **那么** 同步完成后，同级节点必须按 `sort_order` 正确排列

### 需求:新增节点必须自动分配排序值
系统在创建新节点时必须为其分配 `sort_order` 值，使新节点追加到同级节点列表的末尾。

#### 场景:创建根节点分配排序值
- **当** 用户新增一个根节点
- **那么** 系统必须将新节点的 `sort_order` 设为当前所有根节点中最大 `sort_order` 加上间隔值（65536）
- **那么** 如果不存在其他根节点，`sort_order` 必须设为间隔值（65536）

#### 场景:创建子节点分配排序值
- **当** 用户在某个父节点下新增子节点
- **那么** 系统必须将新子节点的 `sort_order` 设为该父节点下所有子节点中最大 `sort_order` 加上间隔值
- **那么** 如果该父节点下无其他子节点，`sort_order` 必须设为间隔值

### 需求:节点排序值计算必须支持中间插入与局部重排
系统在将节点插入两个兄弟节点之间时必须计算中间排序值；当计算出的新值与相邻值差距不足时，系统必须对当前父节点下所有兄弟节点执行局部重排。

#### 场景:在两个节点之间插入
- **当** 节点被移动到两个相邻兄弟节点之间
- **那么** 系统必须将新的 `sort_order` 设为前后兄弟节点 `sort_order` 的中间值

#### 场景:插入到同级列表开头
- **当** 节点被移动到某个父节点的子节点列表开头
- **那么** 系统必须将新的 `sort_order` 设为当前第一个兄弟节点 `sort_order` 减去间隔值

#### 场景:插入到同级列表末尾
- **当** 节点被移动到某个父节点的子节点列表末尾
- **那么** 系统必须将新的 `sort_order` 设为当前最后一个兄弟节点 `sort_order` 加上间隔值

#### 场景:间隔不足触发局部重排
- **当** 计算出的中间值与相邻值差距小于等于 1
- **那么** 系统必须在同一数据库事务中对当前父节点下所有兄弟节点重新分配排序值
- **那么** 重排后的间隔必须恢复为标准间隔值（65536）
- **那么** 重排过程禁止影响其他父节点下的节点

### 需求:排序字段必须通过云同步透传
系统必须将 `sort_order` 字段纳入云同步模型，确保排序值在多端保持一致。

#### 场景:上传节点包含排序值
- **当** 系统将本地节点同步到 Supabase
- **那么** 同步数据必须包含该节点的 `sort_order` 值

#### 场景:下载节点包含排序值
- **当** 系统从 Supabase 拉取节点数据
- **那么** 系统必须将远端的 `sort_order` 值写入本地节点
- **那么** 如果远端 `sort_order` 缺失，系统必须使用默认值 0
