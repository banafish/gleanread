# 规范：settings-appearance

## 目的
定义设置页面中外观设置功能的需求，包括主题模式切换和主题颜色选择，以及偏好的持久化机制。

## 新增需求

### 需求:用户必须能够切换主题模式
系统必须在设置页面外观区域提供浅色、深色、跟随系统三种主题模式的切换功能，且切换后必须立即生效。

#### 场景:选择浅色模式
- **当** 用户在外观设置中选择"浅色"模式
- **那么** 系统必须立即切换为浅色主题
- **那么** 系统必须将选择持久化到 DataStore

#### 场景:选择深色模式
- **当** 用户在外观设置中选择"深色"模式
- **那么** 系统必须立即切换为深色主题
- **那么** 系统必须将选择持久化到 DataStore

#### 场景:选择跟随系统模式
- **当** 用户在外观设置中选择"跟随系统"模式
- **那么** 系统必须根据当前系统暗色模式设置决定使用浅色或深色主题
- **那么** 系统必须在系统模式变化时自动跟随切换

#### 场景:应用启动时恢复主题模式
- **当** 用户启动应用
- **那么** 系统必须从 DataStore 读取上次保存的主题模式并应用
- **那么** 若 DataStore 中没有保存值，系统必须默认使用"跟随系统"模式

### 需求:用户必须能够选择主题颜色
系统必须在设置页面外观区域提供多种预设主题颜色的选择，包括动态色（Material You）和固定色方案，且选择后必须立即生效。

#### 场景:选择动态色
- **当** 用户选择"动态"主题颜色且设备支持 Material You（Android 12+）
- **那么** 系统必须使用设备壁纸提取的动态颜色方案

#### 场景:选择动态色但设备不支持
- **当** 用户选择"动态"主题颜色但设备不支持 Material You（Android 11 及以下）
- **那么** 系统必须回退到默认的固定颜色方案

#### 场景:选择预设主题色
- **当** 用户选择任一预设主题色（海洋、紫色、森林、樱花、琥珀、石墨）
- **那么** 系统必须立即切换到该预设颜色方案
- **那么** 系统必须将选择持久化到 DataStore

#### 场景:应用启动时恢复主题颜色
- **当** 用户启动应用
- **那么** 系统必须从 DataStore 读取上次保存的主题颜色并应用
- **那么** 若 DataStore 中没有保存值，系统必须默认使用"动态"颜色

### 需求:主题模式切换 UI 必须使用可选卡片样式
系统必须以水平排列的可选卡片形式展示浅色、深色、跟随系统三种模式选项，当前选中项必须有明显的视觉区分。

#### 场景:渲染模式选择卡片
- **当** 设置页面外观区域展示主题模式选项
- **那么** 系统必须水平排列浅色、深色、跟随系统三个可选卡片
- **那么** 每个卡片必须包含图标和文字标签
- **那么** 当前选中的卡片必须有明显的选中状态（如边框高亮或背景色变化）

### 需求:主题颜色选择 UI 必须使用圆形色块样式
系统必须以水平排列的圆形色块展示所有可选主题颜色，当前选中项必须显示勾选标记。色块数量超出屏幕宽度时必须支持水平滑动。

#### 场景:渲染颜色选择器
- **当** 设置页面外观区域展示主题颜色选项
- **那么** 系统必须以圆形色块展示所有预设颜色
- **那么** 每个色块下方必须显示颜色名称标签
- **那么** 当前选中的色块上必须显示勾选图标

#### 场景:颜色列表水平滑动
- **当** 预设颜色数量超出屏幕可显示范围
- **那么** 系统必须支持水平滑动浏览所有颜色选项
- **那么** 系统禁止换行显示
