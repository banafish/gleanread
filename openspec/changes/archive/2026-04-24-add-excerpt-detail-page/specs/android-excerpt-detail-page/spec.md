## ADDED Requirements

### 需求:摘录详情页必须提供阅读优先的单条摘录视图
系统必须为单条摘录提供独立详情页，并在默认阅读态中按“元信息、摘录内容、我的想法、来源”的顺序展示内容；页面顶栏必须提供返回动作、标题“摘录详情”和进入编辑态的动作。

#### 场景:打开已归档摘录详情
- **当** 用户从任一支持的工作台入口打开一条已归档摘录
- **那么** 系统必须显示该摘录的详情页
- **那么** 元信息区必须展示该摘录当前挂载的知识树节点、标签和创建时间
- **那么** 页面必须优先展示摘录正文，其次展示想法和来源信息

#### 场景:打开 Inbox 摘录详情
- **当** 用户打开一条未归档的摘录
- **那么** 元信息区必须以 Inbox 标识替代知识树节点
- **那么** 页面仍必须展示该摘录的标签和创建时间

### 需求:摘录详情页必须按内容状态展示想法与来源
系统必须在阅读态中根据摘录内容状态决定想法区和来源区的呈现方式：想法为空时显示弱提示，来源标题与来源链接必须拆分展示，且当两者都为空时禁止显示整个来源区。

#### 场景:想法为空
- **当** 当前摘录的想法内容为空
- **那么** 系统必须在“我的想法”分区显示“还没有想法”之类的弱提示
- **那么** 系统不得用空白区域替代该提示

#### 场景:来源信息为空
- **当** 当前摘录的来源标题和来源链接都为空
- **那么** 系统必须在阅读态隐藏整个来源分区

#### 场景:来源信息存在
- **当** 当前摘录至少存在来源标题或来源链接之一
- **那么** 系统必须把来源标题和来源链接拆开显示，而不得把两者混成单个字段
- **那么** 若来源链接存在，系统必须以单行省略形式展示该链接

### 需求:摘录详情页必须通过选择器编辑标签与挂载位置
系统必须在编辑态允许用户调整标签和挂载位置，并通过标签编辑入口与挂载编辑入口打开选择器；系统不得在元信息区展开多个自由文本输入框替代这些元信息编辑操作。

#### 场景:打开标签编辑
- **当** 用户在编辑态点击标签编辑入口或标签 Chip
- **那么** 系统必须打开可多选的标签选择器，并显示当前已选标签
- **那么** 用户确认后，页面元信息区必须立即显示最新标签集合

#### 场景:打开挂载编辑
- **当** 用户在编辑态点击当前归档节点或 Inbox 入口
- **那么** 系统必须打开挂载位置选择器，并允许用户切换到其他知识树节点或回到 Inbox
- **那么** 用户确认后，页面元信息区必须立即显示最新挂载位置

### 需求:摘录详情页必须支持页面内编辑与本地保存
系统必须允许用户在同一详情页中切换阅读态和编辑态；编辑态必须支持修改摘录正文、想法、来源标题、来源链接、标签和挂载位置，创建时间必须保持只读，且只有在内容实际发生变更时才允许强调或启用保存动作。

#### 场景:进入编辑态
- **当** 用户在阅读态点击编辑动作
- **那么** 系统必须切换到编辑态，并把顶栏标题更新为“编辑摘录”
- **那么** 摘录正文、想法、来源标题和来源链接必须切换为可编辑输入区域
- **那么** 标签和归档位置必须继续显示，并提供显式的选择器入口
- **那么** 创建时间必须继续以只读形式展示

#### 场景:编辑态无变更
- **当** 用户进入编辑态但尚未修改任何字段
- **那么** 系统必须将保存动作显示为禁用或非强调状态

#### 场景:保存摘录编辑
- **当** 用户修改任一可编辑字段或元信息后点击保存
- **那么** 系统必须更新该摘录的本地记录、标签关系和挂载位置，并保留原创建时间不变
- **那么** 系统必须将该摘录变更标记为待同步的本地编辑
- **那么** 系统必须退出编辑态并回到阅读态，且立即显示最新内容

### 需求:摘录详情页必须支持来源跳转
系统必须在阅读态为存在来源链接的摘录提供可点击的来源跳转入口。

#### 场景:点击来源链接
- **当** 用户在摘录详情页点击带有来源链接的来源区
- **那么** 系统必须使用系统链接处理能力打开该来源链接
