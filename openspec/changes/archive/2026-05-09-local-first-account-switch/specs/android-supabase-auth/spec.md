## 修改需求

### 需求:退出登录必须停止同步且不破坏本地数据
系统必须支持用户退出 Supabase 登录。在退出时，系统必须先拦截并检查同步队列状态；退出后停止所有网络同步并安全关闭对应的数据库连接，但不删除对应的 `.db` 文件，同时将状态重置为使用独立的访客数据库。

#### 场景:退出登录时的队列拦截与清理
- **当** 用户退出 Supabase 登录
- **那么** 系统必须先检查当前用户的同步队列中是否存在未上传的本地修改，若有则必须弹窗警告用户
- **那么** 系统必须清除本地 Supabase 会话并断开后台同步及 Realtime 订阅
- **那么** 系统必须关闭当前用户的 `local_data_{user_id}.db` 数据库连接，且禁止删除该文件
- **那么** 系统必须清空 Activity 栈并将用户重置至访客状态（重新绑定至 `guest.db`）
