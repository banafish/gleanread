## 为什么

当前极速摘录在浏览器和微信公众号里主要依赖 `ACTION_SEND` 的 `Intent` 字段获取正文、URL 和标题，但不少“选中文字分享”场景只会传出正文，导致摘录弹窗经常拿不到来源链接和页面标题。随着我们已经确认现有 `UrlExtractor` 覆盖不到这类宿主差异，现在需要补上一条独立于分享 `Intent` 的上下文采集通道，避免快摘体验在高频使用场景里持续降级。

## 变更内容

- 为 Android 端新增“最近页面上下文缓存”能力，在用户仍停留在浏览器或微信公众号页面时，短时缓存最近一次可识别的页面标题、URL、来源应用和采集时间。
- 为极速摘录新增可选的无障碍增强模式，用于在 `Intent` 缺失标题或 URL 时，以最近页面上下文补齐来源信息。
- 为极速摘录新增 `ACTION_PROCESS_TEXT` 文本处理入口，覆盖“选中文字后直接交给 GleanRead”这一系统级入口。
- 调整快摘数据合并规则：优先使用分享 `Intent` 的明确字段，其次回退到最近页面上下文缓存，最后保持手动输入降级路径。
- 打通来源标题持久化链路，使快摘保存时能够把 `sourceTitle` 与 `url` 一并写入本地 Room 并回显到工作台摘录卡片。
- 为无障碍增强模式补充权限引导、不可用状态提示和安全边界，确保该模式是显式开启、可关闭、可降级的辅助能力。

## 功能 (Capabilities)

### 新增功能
- `android-page-context-cache`: 提供短时页面上下文缓存与无障碍增强采集能力，用于为外部快摘补齐最近页面的标题、URL、来源应用和有效期。

### 修改功能
- `fast-capture-widget`: 扩展外部快摘入口，使其同时支持 `ACTION_SEND`、`ACTION_PROCESS_TEXT`，并在 `Intent` 信息不足时消费最近页面上下文缓存。
- `url-display-in-capture-ui`: 调整极速摘录弹窗的来源展示与编辑行为，使其能够显示补齐后的标题和链接，并清晰表达自动补齐与手动输入的状态。
- `android-local-first-knowledge-store`: 扩展本地摘录保存规则，使快摘入口能持久化 `source_title`、`url` 以及与来源补齐相关的必要元数据。

## 影响

- Android Manifest 中的分享入口、文本处理入口与无障碍服务声明。
- `FastCaptureActivity`、快摘来源解析/合并逻辑、Room 持久化与摘录流展示。
- 新增短时页面上下文缓存层、无障碍服务实现、权限引导 UI 与相关测试。
