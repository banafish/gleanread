## 新增需求

### 需求:快摘保存必须持久化来源标题
系统必须在外部快摘解析到来源标题时，将其作为摘录的可选来源元数据持久化到本地库。

#### 场景:保存带来源标题的快摘
- **当** 外部快摘入口已经解析或补齐出 `sourceTitle`
- **那么** 系统必须把该值写入本地 `excerpts` 记录
- **那么** 工作台后续读取该摘录时必须能够回显该来源标题

#### 场景:来源标题缺失时保存
- **当** 外部快摘入口未解析到来源标题
- **那么** 系统必须允许以空的 `sourceTitle` 保存摘录
- **那么** 来源标题缺失不得阻塞摘录写入本地库

## 修改需求

### 需求:Room 本地知识库必须落地四表与同步字段
系统必须在 Android 端使用 Room 持久化 `knowledge_tree_node`、`tags`、`excerpts`、`excerpt_tags` 四张表，并保留 `id`、`user_id`、`is_deleted`、`sync_status` 等离线优先字段；其中 `excerpts` 表必须支持保存可选的 `url` 与 `source_title` 来源字段。

#### 场景:初始化本地数据库
- **当** 应用首次创建本地数据库
- **那么** 系统必须创建上述四张表及其主键和索引，并使新增记录能够携带待同步状态
- **那么** `excerpts` 表必须允许 `url` 与 `source_title` 为空，以兼容不同来源质量的快摘入口

### 需求:保存新的未归档摘录
当用户通过主工作台或快速记录面板新建一条摘录且未选择知识树节点时，系统必须把记录写入 `excerpts` 表、令 `tree_node_id` 为空、把 `sync_status` 标记为待同步，并在存在来源信息时一并保存 `url` 与 `source_title`。

#### 场景:保存新的未归档摘录
- **当** 用户通过主工作台或快速记录面板新建一条摘录且未选择知识树节点
- **那么** 系统必须把记录写入 `excerpts` 表、令 `tree_node_id` 为空、把 `sync_status` 标记为待同步，并为关联标签写入 `excerpt_tags` 关系与本地热度更新
- **那么** 若当前摘录包含来源 URL 或来源标题，系统必须把这些可选来源字段一并保存到 `excerpts` 表
