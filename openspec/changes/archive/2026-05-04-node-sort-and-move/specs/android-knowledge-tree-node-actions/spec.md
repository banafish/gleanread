# 增量规范：android-knowledge-tree-node-actions

## 新增需求

### 需求:知识树节点必须支持长按整张卡片拖拽同级排序交互
系统必须在知识树首页一级节点卡片和分支页节点行支持长按整张卡片进入拖拽同级排序模式。此交互仅支持同级排序移动，不支持跨层级移动，也不支持单独拖拽卡片内部的子节点。此交互与现有的更多菜单中"移动"操作互补：拖拽用于同级快速调整，"移动"菜单用于跨层级移动。

#### 场景:长按一级节点卡片进入拖拽模式
- **当** 用户在知识树首页长按某个一级节点卡片（整张卡片）
- **那么** 系统必须将该节点进入拖拽同级排序模式
- **那么** 系统禁止将长按操作与展开/收起或页面跳转混淆
- **那么** 拖拽仅支持同级排序移动，不支持跨层级移动

#### 场景:长按分支页节点行进入拖拽模式
- **当** 用户在知识树分支页长按某个节点行（整张卡片）
- **那么** 系统必须将该节点进入拖拽同级排序模式
- **那么** 拖拽行为必须与一级节点卡片的拖拽模式一致
- **那么** 拖拽仅支持同级排序移动，不支持跨层级移动

#### 场景:禁止单独拖拽卡片内子节点
- **当** 用户尝试长按卡片内部的二三级子节点进行单独拖拽
- **那么** 系统禁止对卡片内子节点单独触发拖拽
- **那么** 只有长按整张卡片才能进入拖拽模式

## 修改需求

### 需求:知识树必须支持新增根节点与新增子节点
系统必须支持从知识树首页新增根节点，也必须支持从知识树分支页或节点更多菜单新增子节点；新增节点必须通过对话框输入名称，并在保存后立即反映到当前知识树页面。新增节点必须自动分配 `sort_order` 排序值，追加到同级节点列表末尾。

#### 场景:首页新增根节点
- **当** 用户在知识树首页点击新增根节点 FAB 或空状态中的"新增节点"入口
- **那么** 系统必须打开新增节点对话框
- **那么** 用户保存后系统必须创建新的一级节点并立即刷新知识树首页
- **那么** 新节点的 `sort_order` 必须为当前根节点列表中最大值加上间隔值

#### 场景:分支页新增当前节点子节点
- **当** 用户在某个知识树分支页点击"新增子节点" FAB
- **那么** 系统必须打开新增子节点对话框，并默认将当前分支节点作为父节点
- **那么** 用户保存后系统必须把新节点显示在当前分支页的子节点列表末尾
- **那么** 新节点的 `sort_order` 必须为当前父节点下最大值加上间隔值

#### 场景:从节点菜单新增任意子节点
- **当** 用户在任意知识树节点的更多菜单中点击"新增子节点"
- **那么** 系统必须打开新增子节点对话框，并显示所选节点作为父节点
- **那么** 用户保存后系统必须把新节点追加到该父节点下的子节点列表末尾
- **那么** 新节点的 `sort_order` 必须为该父节点下最大值加上间隔值
